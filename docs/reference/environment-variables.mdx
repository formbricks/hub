## Required Variables[‚Äã](#required-variables "Direct link to Required Variables")

### `SERVICE_DATABASE_URL`[‚Äã](#service_database_url "Direct link to service_database_url")

PostgreSQL connection string.

**Format:**

```
postgres://[user]:[password]@[host]:[port]/[database]?[params]
```

**Examples:**

```
# Local developmentSERVICE_DATABASE_URL=postgres://formbricks:formbricks_dev@localhost:5432/hub_dev?sslmode=disable# Production with SSLSERVICE_DATABASE_URL=postgres://user:pass@db.example.com:5432/hub_prod?sslmode=require# With connection poolingSERVICE_DATABASE_URL=postgres://user:pass@pooler.example.com:5432/hub_prod?pool_max_conns=20
```

**Default:** None (required)

***

## Server Configuration[‚Äã](#server-configuration "Direct link to Server Configuration")

### `SERVICE_PORT`[‚Äã](#service_port "Direct link to service_port")

Port for the HTTP server.

**Examples:**

```
SERVICE_PORT=8080  # DefaultSERVICE_PORT=3000  # Custom port
```

**Default:** `8080`

***

### `SERVICE_HOST`[‚Äã](#service_host "Direct link to service_host")

Host address to bind to.

**Examples:**

```
SERVICE_HOST=0.0.0.0  # Listen on all interfaces (default)SERVICE_HOST=127.0.0.1  # Localhost onlySERVICE_HOST=10.0.1.5  # Specific IP
```

**Default:** `0.0.0.0`

<Warning>
  For production, use `0.0.0.0` to allow external connections. For local development, either works.
</Warning>

***

## Security[‚Äã](#security "Direct link to Security")

### `SERVICE_API_KEY`[‚Äã](#service_api_key "Direct link to service_api_key")

Optional API key for authentication. If set, all API requests (except `/health` and `/docs`) require the `X-API-Key` header.

**Examples:**

```
# No authentication (development)# SERVICE_API_KEY=  # Commented out or empty# With authentication (production)SERVICE_API_KEY=your-secret-key-here
```

**Default:** Empty (authentication disabled)

**Usage:**

```
curl -H "X-API-Key: your-secret-key-here" \  http://localhost:8080/v1/experiences
```

[Learn more about authentication ‚Üí](/core-concepts/authentication)

## Webhooks[‚Äã](#webhooks "Direct link to Webhooks")

### `SERVICE_WEBHOOK_URLS`[‚Äã](#service_webhook_urls "Direct link to service_webhook_urls")

Comma-separated list of webhook URLs to receive experience data events.

**Examples:**

```
# Single webhookSERVICE_WEBHOOK_URLS=https://api.example.com/webhooks/hub# Multiple webhooksSERVICE_WEBHOOK_URLS=https://api.example.com/webhooks,https://analytics.example.com/events# No webhooksSERVICE_WEBHOOK_URLS=  # Empty
```

**Default:** Empty (no webhooks)

**Events sent:**

* `experience.created`
* `experience.updated`
* `experience.deleted`

[Learn more about webhooks ‚Üí](/core-concepts/webhooks)

## AI Features[‚Äã](#ai-features "Direct link to AI Features")

### `SERVICE_OPEN_AI_KEY`[‚Äã](#service_open_ai_key "Direct link to service_open_ai_key")

OpenAI API key for AI-powered features (enrichment and embeddings). Required for both enrichment and semantic search.

**Examples:**

```
# With OpenAI featuresSERVICE_OPEN_AI_KEY=sk-proj-xxxxxxxxxxxxxxxxxxxxx# Without AI featuresSERVICE_OPEN_AI_KEY=  # Empty
```

**Default:** Empty (AI features disabled)

<Info>
  Get your OpenAI API key from [platform.openai.com/api-keys](https://platform.openai.com/api-keys)
</Info>

***

### `SERVICE_OPENAI_ENRICHMENT_MODEL`[‚Äã](#service_openai_enrichment_model "Direct link to service_openai_enrichment_model")

OpenAI chat model for sentiment, emotion, and topic analysis.

**Recommended Models:**

* `gpt-4o-mini` - Fast and cost-effective (default)
* `gpt-4o` - Higher accuracy, more expensive
* `gpt-5-mini` - Latest model (when available)

**Examples:**

```
SERVICE_OPENAI_ENRICHMENT_MODEL=gpt-4o-mini  # Default, recommendedSERVICE_OPENAI_ENRICHMENT_MODEL=gpt-4o       # Higher quality
```

**Default:** `gpt-4o-mini`

**Enabled when:** `SERVICE_OPEN_AI_KEY` is set and this value is non-empty.

[Learn more about AI enrichment ‚Üí](/core-concepts/ai-enrichment)

***

### `SERVICE_OPENAI_EMBEDDING_MODEL`[‚Äã](#service_openai_embedding_model "Direct link to service_openai_embedding_model")

OpenAI embeddings model for semantic search (vector generation).

**Recommended Models:**

* `text-embedding-3-small` - Cost-effective, 1536 dimensions (recommended)
* `text-embedding-3-large` - Higher accuracy, 3072 dimensions, 6.5x cost

**Cost Comparison (per 1M tokens):**

| Model                    | Cost  | Dimensions | Use Case                       |
| ------------------------ | ----- | ---------- | ------------------------------ |
| `text-embedding-3-small` | $0.02 | 1536       | General feedback (recommended) |
| `text-embedding-3-large` | $0.13 | 3072       | Technical/complex content      |

**Examples:**

```
SERVICE_OPENAI_EMBEDDING_MODEL=text-embedding-3-small  # RecommendedSERVICE_OPENAI_EMBEDDING_MODEL=text-embedding-3-large  # Higher accuracy
```

**Default:** `text-embedding-3-small`

**Enabled when:** `SERVICE_OPEN_AI_KEY` is set and this value is non-empty.

<Check>
  For customer feedback and survey responses, `text-embedding-3-small` provides excellent results at 1/6th the cost of the large model.
</Check>

[Learn more about semantic search ‚Üí](/core-concepts/ai-enrichment#embeddings-and-semantic-search)

***

### `SERVICE_ENRICHMENT_TIMEOUT`[‚Äã](#service_enrichment_timeout "Direct link to service_enrichment_timeout")

Timeout in seconds for AI API calls (both enrichment and embeddings).

**Examples:**

```
SERVICE_ENRICHMENT_TIMEOUT=10  # DefaultSERVICE_ENRICHMENT_TIMEOUT=30  # For slower connections
```

**Default:** `10`

***

### `SERVICE_ENRICHMENT_WORKERS`[‚Äã](#service_enrichment_workers "Direct link to service_enrichment_workers")

Number of concurrent background workers processing AI jobs.

**Examples:**

```
SERVICE_ENRICHMENT_WORKERS=3   # DefaultSERVICE_ENRICHMENT_WORKERS=10  # High volumeSERVICE_ENRICHMENT_WORKERS=1   # Low volume / rate limiting
```

**Default:** `3`

<Warning>
  OpenAI has rate limits. Adjust workers based on your tier:

  * Free tier: 3 requests/minute ‚Üí use 1 worker
  * Tier 1: 500 requests/minute ‚Üí use 3-5 workers
  * Tier 2+: Higher limits ‚Üí use 10+ workers
</Warning>

***

### `SERVICE_ENRICHMENT_POLL_INTERVAL`[‚Äã](#service_enrichment_poll_interval "Direct link to service_enrichment_poll_interval")

Seconds between worker queue polls.

**Examples:**

```
SERVICE_ENRICHMENT_POLL_INTERVAL=1  # Default, responsiveSERVICE_ENRICHMENT_POLL_INTERVAL=5  # Lower polling frequency
```

**Default:** `1`

## Logging[‚Äã](#logging "Direct link to Logging")

### `SERVICE_LOG_LEVEL`[‚Äã](#service_log_level "Direct link to service_log_level")

Logging verbosity level.

**Options:**

* `debug` - Detailed debugging information
* `info` - General informational messages (recommended)
* `warn` - Warning messages only
* `error` - Error messages only

**Examples:**

```

```

**Default:** `info`

## Troubleshooting[‚Äã](#troubleshooting "Direct link to Troubleshooting")

### "DATABASE\_URL is required" Error[‚Äã](#database_url-is-required-error "Direct link to \"DATABASE_URL is required\" Error")

**Problem:** Missing `SERVICE_DATABASE_URL` environment variable.

**Solution:** Set the variable in your `.env` file:

```
SERVICE_DATABASE_URL=postgres://formbricks:formbricks_dev@localhost:5432/hub_dev?sslmode=disable
```

### "Cannot connect to database" Error[‚Äã](#cannot-connect-to-database-error "Direct link to \"Cannot connect to database\" Error")

**Problem:** Database connection details are incorrect or database is not running.

**Solutions:**

1. Check PostgreSQL is running: `docker-compose ps`
2. Verify connection string format
3. Test connection manually:
   ```
   psql "postgres://formbricks:formbricks_dev@localhost:5432/hub_dev?sslmode=disable"
   ```

### API Key Not Working[‚Äã](#api-key-not-working "Direct link to API Key Not Working")

**Problem:** Getting 401 errors despite setting `X-API-Key` header.

**Solutions:**

1. Verify `SERVICE_API_KEY` is set: `grep SERVICE_API_KEY .env`
2. Check header name is exact: `X-API-Key` (case-sensitive)
3. Ensure key matches exactly (no extra spaces)
